排序的基本概念

内排序和外排序

内排序：排序的数据量不是很大，所有数据可以存到内存中。

内部排序一般采用顺序存储结构效率更高。

外排序：排序的数据量很大，数据需要存到外存中。

算法的稳定性

稳定：相同值的元素之间的相对位置在排序前后不会发生改变
不稳定：相同值的元素之间的相对位置在排序前后可能发生改变

散列（Hash）表

散列表主要是指将一个集合映射到0、1、2…的索引上，为了便于查找。
查找效率 == 时间复杂度 存储效率 == 负载因子

负载因子

$\text{负载因子} = \frac{\text{已有的元素数}}{\text{数组长度}}$
Hash表中一般每个索引对应一个数组，然后数组中存储对应元素，这个数组中存储元素个数决定了查找效率，即负载因子越小、查找效率越高。

查找的基本概念

平均查找长度ASL

平均查找长度 ASL = 每个元素 查找概率 * 找到第i个元素需要进行的比较次数 的和。
ASL（成功） = $\sum_i{P_iC_i}$，其中每个元素查找概率$P_i = 1/n$，$C_i$表示查找成功需要的比较次数
ASL（失败） = $\sum_{\text{区间}}{P_iC_i}$，其中$P_i$表示查找区间找到的概率，所以$P_i=1/(n+1)$

最小生成树MST

有向图、无向图都有最小生成树，不过一般考研只考无向图的最小生成树。
生成树定义1：从图中选n-1条边使得整个图连通，同时总边权和最小。
生成树定义2：从图中选n-1条边使得整个图连通，同时最大边权最小。
以上两种定义方式求出来的生成树本质上是相同的。
