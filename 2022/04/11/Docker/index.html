Docker概述

Docker为什么出现

  1. 一款产品：开发和上线，需要两套环境。
  2. 开发 — 运维 问题：我在我的电脑上可以运行。
  3. 版本更新导致服务不可用。
  4. 环境配置麻烦，每个机器都需要部署环境。 Docker实现：项目带上环境进行打包（镜像）。
    传统：开发做jar包，运维来来部署
    现在：开发打包部署一套流程完成

    Docker核心思想

    Docker是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的镜像中，然后发布到流行的Linux或Windows机器上，也可以实现虚拟化。容器完全使用沙箱机制，相互之间不会有任何接口。
    Docker的思想来自于集装箱。之前的多个项目（或应用）都是运行在一个环境下（例如JRE），可能出现各种冲突（例如端口冲突等）。而Docker的核心思想就是==隔离==，通过镜像（或者说容器）来实现每个应用进行分别打包装箱，使得他们彼此隔离，使得服务器的利用率达到极致。

    Docker历史

    2010年，在美国成立了dotCloud，一开始做一些PaaS的云计算服务，类似于Linux有关的容器技术。他们将自己的技术（容器化技术）命名为Docker。
    2013年，Docker因为没有引起行业注意，所以后来开源了。然后其他人发现了Docker的优点，社区非常活跃。
    2014年，Docker 1.0发布。

    Docker为什么这么火

    相对于虚拟机技术（Vmware、Oracle Box），十分轻巧。
  5. 虚拟机也属于虚拟化技术，也是隔离，每个镜像都是一个完整的系统。
  6. Docker容器技术也是一种虚拟化技术，但是只打包最核心的环境，十分轻巧。

    相关网站

    文档地址：https://docs.docker.com/
    仓库地址：https://hub.docker.com/

    Docker可以做什么

虚拟机技术缺点：

  1. 完整的操作系统，资源占用非常多
  2. 冗余步骤很多
  3. 启动很慢
  4. 所有APP运行在同一个环境下，如果环境出现问题，所有APP都不能运行

容器化技术：

  1. 并不是模拟完整的操作系统，只模拟核心内容
  2. 每个APP运行在独立的容器中，拥有独立的资源（Lib库等），即使其中一个容器出现故障，其他APP运行不收影响

比较Docker和虚拟机技术的不同：

  1. 传统虚拟机，虚拟出一套硬件，运行一个完整的操作系统，然后在这个操作系统上安装和运行软件。
  2. 而容器内的应用直接运行在宿主机的内核上，容器是没有自己的内核的，也没有虚拟硬件。容器间是相互隔离的，每个容器内都有一个属于自己的文件系统，互不影响。

DevOps（开发、运维）

  1. 应用更快速的交付和部署
    1. 传统：一堆帮助文档，安装程序
    2. Docker：打包镜像发布测试，一键运行
  2. 更便捷的升级和扩缩容
    1. 使用Docker之后，部署应用就类似搭积木一般
  3. 更简单的系统运维
    1. 在容器化之后，开发和测试环境都是高度相似的
  4. 更高效的计算资源利用
    1. Docker是内核级别的虚拟化，可以在一个物理机上运行很多的容器实例，服务器性能应用到极致。

Docker安装

Docker的基本组成


Docker主要有：客户端（Client）、服务器（Docker_Host），仓库（Registry）
在客户端执行Docker命令之后，相应的命令会在服务器的守护进程（Docker Daemon）上进行执行，守护进程需要通过仓库（repository）镜像（Images）来创建相应的容器（Container）进行运行程序。
仓库（repository）
仓库主要来存储镜像。主要分为公有仓库和私有仓库。
镜像（image）
Docker镜像就好比是一个模板，可以通过这个模板来创建容器来分别提供服务，一个镜像可以创建多个容器，最终服务运行或者项目运行就是在这个容器中进行的。
容器（container）
Docker利用容器技术，独立运行一个或者一组应用，通过镜像来创建。每个容器相当于一个微型的操作系统。

底层原理

Docker是怎么工作？

Docker是一个Client - Server结构的系统，Docker的守护进程运行在主机上，通过Socket从客户端访问。
Docker-Server接收到Docker-Client的指令，就会执行这个命令。

为什么Docker比虚拟机快

  1. Docker有着比虚拟机更少的抽象层
  2. Docker利用的是宿主机的内核，VM需要是Guest OS。所以新建一个容器的时候，Docker不需要像虚拟机一样重新加载一个新的操作系统内核。

    虚拟技术分为全虚拟、半虚拟、容器技术三类

docker run

OPTIONS

--name 指定容器名字，用以区分容器
-d 后台运行
-it 使用交互方式运行，进入容器查看内容
-p 指定容器端口，同时可以将端口和主机端口进行映射

  • -p ip:主机port:容器port
  • -p 主机port:容器port
  • -p 容器port
  • 容器port
    -P 随机指定容器端口

    例

-d 表示后台运行
--name 指定容器名字
-p 指定暴露端口，“主机端口:容器端口”实现容器端口到主机端口的映射

--rm 表示用完这个容器之后就自动删除，一般用于测试

-e 可以对运行的容器添加一些限制，例如ES_JAVA_OPTS=”-Xms64m -Xmx512m”添加运行内存限制。

启动和停止容器

镜像相当于是一个类模板，容器相当于是一个实例。采用docker run IMAGE创建容器以后，如果不删除容器，这个容器是一直存在的，对其的更改也是一直保存的，而对一个容器的更改不会影响镜像和其他容器的数据。

常用命令

后台启动容器

docker run -d IMAGES

常见的坑：通过-d的形式创建容器时，结果会发现容器直接停止运行了。
原因：Docker容器使用后台运行，就必须要有一个前台进程，Docker发现没有应用就会自动停止。因为容器启动后，会发现自己没有提供服务，所以就会立即停止。
