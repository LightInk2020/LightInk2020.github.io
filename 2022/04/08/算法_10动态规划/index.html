理论

动态规划，英文：Dynamic Programming，简称DP，如果某一问题有很多重叠子问题，使用动态规划是最有效的。
所以动态规划中每一个状态一定是由上一个状态推导出来的，这一点就区分于贪心，贪心没有状态推导，而是从局部直接选最优的，

动态规划解题步骤

  1. 确定dp数组（dp table）以及下标的含义
  2. 确定递推公式
  3. dp数组如何初始化
  4. 确定遍历顺序
  5. 举例推导dp数组

#好题

卡特兰数解法

#注意 C++中*=、+=、-=、\=的运算优先级很低，基本是等式右边其他运算符进行完成以后才会进行。所以如果等式右边有/要考虑是否采用这种运算符，还是把运算符移到等号右边。
如：rs = 2; rs *= 10/4; 得到的结果4可能和预想的5不同。
#知识点 ==卡特兰数==可以直接计算n个结点所对应的二叉树的形态数。也就是二叉搜索树的形态（确定的前序遍历次序）。

动态规划解法



来看看n为3的时候，
当1为头结点的时候，其右子树有两个节点，这两个节点的布局和n为2的时候两棵树的布局是一样的。
（我们是求不同树的数量，并不用把搜索树都列出来，所以不用关心其具体数值的差异）
当3为头结点的时候，其左子树有两个节点，这两个节点的布局和n为2的时候两棵树的布局是一样的。
当2为头结点的时候，其左右子树都只有一个节点，布局和n为1的时候只有一棵树的布局是一样的。
发现到这里，其实我们就找到了重叠子问题了，其实也就是发现可以通过dp[1] 和 dp[2] 来推导出来dp[3]的某种方式。
dp[3]，就是元素1为头结点搜索树的数量 + 元素2为头结点搜索树的数量 + 元素3为头结点搜索树的数量
元素1为头结点搜索树的数量 = 右子树有2个元素的搜索树数量 * 左子树有0个元素的搜索树数量
元素2为头结点搜索树的数量 = 右子树有1个元素的搜索树数量 * 左子树有1个元素的搜索树数量
元素3为头结点搜索树的数量 = 右子树有0个元素的搜索树数量 * 左子树有2个元素的搜索树数量
有2个元素的搜索树数量就是dp[2]。
有1个元素的搜索树数量就是dp[1]。
有0个元素的搜索树数量就是dp[0]。
所以dp[3] = dp[2] * dp[0] + dp[1] * dp[1] + dp[0] * dp[2]

**dp[i] ： 1到i为节点组成的二叉搜索树的个数为dp[i]**。

目标和

494. 目标和 - 力扣（LeetCode） (leetcode-cn.com)
#好题 #技巧
思路
本题同样可以转换为01背包问题，物品是带选取符号的所有数，但是由于待选取符号的包含正数负数，而背包问题全为正数，所以需要转换一下思路。
同样可以将本问题看成选取物品，即认为符号为正数的为被选取出的数，那么背包就变成目标和target对应的正数的和（未被选取的数为负号）。
可以有以下关系推导：
设nums中所有数之和为sum，而对应target的所有正号的数之和为addSum，所有负号的数之和为subSum。则有：
target = addSum - subSum = addSum - (sum - addSum)
addSum = (target + sum) / 2
这样，就得到了正数和，问题就转换为01背包问题。
注意

  1. ==看到(S + sum) / 2 应该担心计算的过程中向下取整有没有影响。==
    例如sum 是5，S是2的话其实就是无解的，所以：
  2. 同时如果 S 的绝对值已经大于sum，那么也是没有方案的。

    这种情况需要考虑，不然求出来的addSum可能是负数。

组合总和 IV

377. 组合总和 Ⅳ - 力扣（LeetCode） (leetcode-cn.com)
#好题
如果把遍历nums（物品）放在外循环，遍历target的作为内循环的话，举一个例子：计算dp[4]的时候，结果集只有{1,3} 这样的集合，不会有{3,1}这样的集合，因为nums遍历放在外层，3只能出现在1后面！
所以本题遍历顺序最终遍历顺序：target（背包）放在外循环，将nums（物品）放在内循环，内循环从前到后遍历。
#知识点
==如果求组合数就是外层for循环遍历物品，内层for遍历背包。
如果求排列数就是外层for遍历背包，内层for循环遍历物品。==

巧妙思维：断开环转换为两个子问题
去掉其中一个房子（即这个房子不偷）就可以将环断开，所以结果集可以分为以下两种：

  1. i = 0 这个房子不偷
  2. i = n 这个房子不偷
