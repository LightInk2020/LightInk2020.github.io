笔试

Huffman编码和Huffman树

Huffman编码

前缀编码

前缀编码是指对字符集进行编码时，要求字符集中任一字符的编码都不是其它字符的编码的前缀。
编码的总长度 = SUM（被编码的字符对应的编码长度 * 字符出现次数有关）

当字符集实现了前缀编码，那么解码结果是唯一的不会有歧义。
前缀编码 <=>所有编码均对应树的叶节点

树的带权路径长度(WPL)

带权路径长度 = SUM（边长度 * 叶节点权重）
在前缀编码中，叶节点权重为字符出现次数，树的带权路径长度等于编码的总长度。

满足WPL最小的树的性质
  1. 所有结点的度数必不为1
  2. 一定存在一个最优解（WPL最小树），使得权最小的两个点互为兄弟
  3. 权值最小的点一定在最下边

    Huffman树

    构造过程

    从下往上构建，每次选择权重最小的两个结点作为兄弟节点，根节点的权重为两个结点的权重和，不断迭代，即可构建Huffman树。

    应用

    Huffman可以应用于每次合并两个、然后合并代价为线性的问题，例如合并果子。
    148. 合并果子 - AcWing题库 #知识点 堆实现
    priority_queue 默认是大根堆，优先向外弹出最大值
    priority_queue<int> q
    priority_queue<int, vector<int>, greater<int>> b; // 小根堆，优先弹出最小值
    149. 荷马史诗 - AcWing题库
    总共需要合并次数m满足，如果恰好能够合并完：
    (k - 1)m + k = n 即 (k - 1)(m + 1) = n -1
    #注意 如何处理n-1不能被k-1整除的问题？
    因为，数的编码要尽可能地短，所以如果不处理n-1不能被k-1整除的问题直接从下向上进行合并，最后一次合并时数的个数小于k，这会导致结果不是最优解。此时合并次数与数的个数关系：
    (k - 1)(m + 1) = n - 1 + d
    d = (k - 1) - (n - 1) % (k - 1)
    解决方案：不足k-1的整数倍的部分用权值为0的数补齐，这样既不影响最后结果，也能实现最优解。

    二进制不需要考虑不能被整除问题，因为最后结果为0或1个数不用再合并。

    2013-4

    题目：已知三叉树T中6个叶节点的权分别是2、3、4、5、6、7，T的带权（外部）路径长度最小是
    46
    #注意 注意题目要求：三叉树
    Huffman编码对于k叉树进行时，如果(k-1|n - 1)不成立，需要先补零使他成立！

    2020-42

    #知识点 如何判定某个字符集的0/1不等长编码是否具有前缀特性？
    采用二叉树/哈夫曼树保存其不等长编码，只需要判断存储有字符信息的节点是否全部为叶节点即可。
    若存储有某个字符信息的节点非叶节点，即有子节点，那么它的0/1编码一定是它孩子节点0/1编码的前缀，违反了前缀特性。
