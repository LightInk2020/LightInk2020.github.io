笔试

散列（Hash）表

散列表主要是指将一个集合映射到0、1、2…的索引上，为了便于查找。
查找效率 == 时间复杂度 存储效率 == 负载因子

负载因子

$\text{负载因子} = \frac{\text{已有的元素数}}{\text{数组长度}}$
Hash表中一般每个索引对应一个数组，然后数组中存储对应元素，这个数组中存储元素个数决定了查找效率，即负载因子越小、查找效率越高。

哈希函数

除余法

H(key) = key % p
或 H(key) = key % p + c 这里p为小于m的质数；
余数总在 0 ～ p-1 之间。

为什么p选取为质数：因为如果p非质数，则key%p得到的结果取值是不均衡的，会导致有些空间被使用概率较小，最后空间利用率下降、发生冲突可能性较大。例如p为偶数，则偶数%偶数一定是偶数，奇数%偶数一定是奇数。

乘余取整法

h(x) = floor(n * ((A * x)的小数部分))，0<A<1，n是hash表数组长度

平方取中法

先平方，然后取中间几位

基数转换法

换成其他进制，然后取其中几位

ELFhash字符串

解决冲突的方式

主要解决不同的元素映射到相同位置的问题。

开散列方法（拉链法）

如果一个索引对应多个数值，则用一个数组或链表存储当前索引对应的元素，然后用索引指向这个数组或链表。
经验上：上机题目中有n个元素要存储到数组中，则开2n大小的数组来存储数据。即保证负载因子大小为0.5，一般效率会比较高。

聚集和二级聚集

聚集：不同的hash值在进行探查时容易探查到同一部分的位置，即一直探查到相同部分，而其他部分空闲。
二次聚集：当表中i,i+1,i+2位置上已经填有记录时，下一个哈希地址为i,i+1,i+2和i+3的记录都将填入i+3的位置，这种在处理冲突过程中发生的两个第一个哈希地址不同的记录争夺同一个后继哈希地址的现象称为“ 二次聚集”，即在处理同义词的冲突过程中又添加了非同义词的冲突。

探查方法

d(0) = hash(x)

  1. 线性探查法 d(i) = (d(0) + i * c) % M。易产生聚集问题。
  2. 二次探查法。易产生二级聚集问题。
    $$
    d(2i - 1) = (d(0) + i^2) % M
    $$
    $$
    d(2i) = (d(0) - i ^2) % M
    $$
  3. 随机探查法。易产生二级聚集问题。
    因为实际上是伪随机数，所以也会产生二次聚集。
  4. 双散列探查法
    $$
    d(i) = (d(0) + i * ReHash(key) ) % m, i =1, 2, …, m-1.
    $$

    AcWing 3820. 未出现过的最小正整数 - AcWing

    朴素写法

    优化写法

    分析：
    因为nums中只有n个数，所以未出现的正整数只有两种情况。一种是nums中的数正好是1、2、…、n，然后未出现的正整数为n + 1；另一种情况是nums中的数有一部分数值范围不在[1, n]之间，那么未出现的正整数取值范围为[1, n]。
    所以只要开大小为n的bool数组，如果1到n的数i在nums中出现就将bool数组对应位置为true，否则默认为false，然后最后输出bool数组中第一个为false的数即可，如果没有为false的，说明未出现的正整数为n + 1。

    查找效率

    用平均搜索长度ASL (Averagy Search Length)衡量散列方法的搜索性能。
    根据搜索成功与否，它又有搜索成功的平均搜索长度ASLsucc和搜索不成功的平均搜索长度ASLunsucc之分。

    搜索成功的平均搜索长度

    搜索成功的平均搜索长度 ASLsucc 是指搜索到表中已有表项的平均探查次数。它是找到表中各个已有表项的探查次数的平均值。

    搜索不成功的平均搜索长度

    搜索不成功的平均搜索长度 ASLunsucc 是指在表中搜索不到待查的表项，但找到插入位置的平均探查次数。它是表中所有可能散列到的位置上要插入新元素时为找到空位置的探查次数的平均值。
    1. 针对的是所有可能散列到的位置，所以所有情况应该包括所有Hash值，默认每种情况概率相同；
    2. 找到空位置为止，如果碰到冲突则需要对应探查方式来继续探查直到找到空位置。

    字符串模式匹配

    简单模式匹配算法

    设目标T的长度为n,模式P 的长度为m,在最坏情况下，比较次数：(n-m+1) * m
    在多数情况下，m远小于n, 因此算法的最坏的时间复杂性为$O(n * m)$。

    KMP算法

    无回溯算法。KMP匹配算法的时间复杂度O(m+n)。
  • 计算特征数的时间复杂度为O(m) 。

    KMP虽然有双重循环，但是时间复杂度是O(n)

前缀子串：模式串P开头的前k个字符，即$p_0, p_1, …, p_{k-1}$
i位置的后缀子串：在P第i位置的左边，取出k个字符，即$p_{i-k+1}, … , p_{i}$
第i位的最长前缀串：与i位置后缀子串相等的最长前缀子串。（非平凡的）

这个串本身即是自己的前缀子串又是自己的后缀子串，这个前后缀称为平凡的前后缀，不在第i位的最长前缀串这个概念中考虑。例如abcabcd中第5位c的最长前缀串是abc，而不是abcabc。
