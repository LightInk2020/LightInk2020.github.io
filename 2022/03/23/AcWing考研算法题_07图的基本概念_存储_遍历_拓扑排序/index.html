AcWing考研算法题-07图的基本概念、存储、遍历、拓扑排序

笔试

图的基本概念

有向图、无向图

树是特殊的图。
图G=<V, E>。
有向图：边有方向的图
无向图：边没有方向的图。很多算法中，无向边可以看作两个有向边进行处理。

混合图：有的边有方向，有的边没有方向构成的图。

度数（出度、入度）

无向图中，度数指节点连接的边的数量。
有向图中，度数等于入度加出度。出度指从节点出发指向其他节点的边数，入度指从其他节点出发指向该节点的边数。

简单图

简单图：没有自环（不存在顶点到其自身的边），没有重边（且同一条边不重复出现）。

路径、环、简单路径

路径
环：路径起点和终点一致
简单路径：路径中不包含重复节点

无向完全图

任意两个顶点之间都存在边，有n个顶点的无向完全图有 n × (n - 1) / 2条边

有向完全图

任意两个顶点之间都存在方向相反的两条弧，有n个顶点的无向完全图有 n × (n - 1) 条弧

稀疏图&稠密图

有很少条边或弧的图称为稀疏图，反之称为稠密图，相对的概念。

图的存储及基本操作

邻接矩阵

适用于稠密图（点数较少的稀疏图也可以用邻接矩阵存储），可存有向图（对称矩阵）、无向图。常用。空间复杂度：$O(n^2)$。无法存重边。
适用算法：Floyd算法等

邻接表

邻接表：每个节点存储一个包含可以到达的节点的集合（无向图中即两点间有边，有向图中即有该点指向对应点的边）。存储方式：单链表、vector、数组、set、queue、stack等。
适用于稀疏图（稠密图也可以用，空间复杂度$O(n^2)$），可存有向图、无向图。常用。空间复杂度：$O(n + m)$。可存重边。

邻接表可以存储重边。

邻接表比较常用，因为空间复杂度相对小。

邻接多重表

邻接表存储无向图时，删除边不容易操作，因为要删除两个存储的数据，比如1到3之间删除边，既要删除1中存储的3，还要删除3中存储的1。但是已知一个边找反向边是困难的，所以邻接多重表针对这个问题进行了改进，比较方便找反向边。
适用于稀疏图，可存无向图。不常用。空间复杂度：$O(n + m)$。可存重边。

十字链表

十字链表是对邻接矩阵的优化。

实际上，十字链表是对矩阵的操作，对矩阵进行压缩存储。

适用于稀疏图，可存有向图、无向图。不常用。空间复杂度：$O(n + m)$。无法存重边。

十字链表常用于Dancing Linksing算法，Dancing Linksing求解数独、八皇后等精确覆盖或重复覆盖类型的问题。

三元组表

适用于稀疏图，可存有向图，无向图。空间复杂度：$O(m)$。可存重边。
常用于Bellman-Ford算法、Kruskal算法。

图的遍历

深度优先搜索DFS

邻接表存储的时间复杂度：$O(n + m)$。邻接矩阵存储的时间复杂度：$O(n^2)$
DFS的到达顺序称为DFS序列。递归在开头写。

DFS的回溯顺序是拓扑排序。递归在结尾写。

#注意 DFS时注意遍历所有点，以防遗漏孤立点。
#注意 因为DFS实现是通过递归实现的，所以需要用到大量的栈空间。但是有的OJ平台栈空间设定的很小，所以有可能出现问题。因此拓扑排序问题尽量用BFS。

广度优先搜索BFS

邻接表存储的时间复杂度：$O(n + m)$。邻接矩阵存储的时间复杂度：$O(n^2)$

无论是DFS还是BFS进行遍历的时候都要记得判重。

#知识点 BFS第一次找到节点一定是通过最短路径找到的，所以BFS可以用来求最短路。

拓扑排序

若一个由图中所有点构成的序列A满足：对于图中的每条边(x,y)，x在A中都出现在y之前，则称A是该图的一个拓扑序列。
一个图的拓扑排序==可能==不唯一。
不是所有图都有拓扑排序，可能有的图没有拓扑排序（例如图内有环）。有向图存在拓扑排序 充要条件 这个图没有环。可以有重边，但是不能有自环。
拓扑排序既可以用DFS实现，也可以用BFS实现。
#注意 因为DFS实现是通过递归实现的，所以需要用到大量的栈空间。但是有的OJ平台栈空间设定的很小，所以有可能出现问题。因此拓扑排序问题尽量用BFS。

BFS实现拓扑排序

思路

  1. 先将所有入度为0的点放入队列
  2. 取出队列里的一个点，访问一下，再将这个点和相连的边从图中删去
  3. 如果这点去除以后他的相邻节点中出现了入度为0的点，则也放入队列
  4. 循环2、3操作直到队列为空即可实现拓扑排序
    时间复杂度：O(n + m)

    判定图是否存在拓扑排序

    通过上述遍历思路进行遍历，然后判断是否所有节点都被遍历过，如果存在节点没有被遍历过，说明图不存在拓扑排序。
    AcWing 848. 有向图的拓扑序列 - AcWing

    DFS实现拓扑排序

    思路

    DFS只需要将访问操作放到dfs函数的尾部输出的即是拓扑排序的逆序。
    理解：将访问操作放到dfs递归入口的后面，说明当前元素的访问将会在他可以深度遍历到的元素的后面执行，这样与拓扑排序的说法正好相反。

    判定图是否存在拓扑排序

    有向图存在拓扑排序 充要条件 这个图没有环。DFS可以判断图中是否存在环，即递归过程中访问到同一个元素即说明图中有环。

    考题

    2011-8、2012-5、2012-6、2013-7、2013-8、2014-7、2015-5、2016-6、2016-7、2017-3、2017-7、2018-7、2020-6

#知识点 时/空复杂度

算法 描述 时间复杂度 空间复杂度
深度优先遍历/拓扑排序 邻接矩阵 $O(n^2)$ $O(n^2)$
深度优先遍历/拓扑排序 邻接表 $O(n+m)$ $O(n+m)$
广度优先遍历/拓扑排序 邻接矩阵 $O(n^2)$ $O(n^2)$
广度优先遍历/拓扑排序 邻接表 $O(n+m)$ $O(n+m)$
#技巧 广度优先遍历得到的序列一定是按照最短距离排序的，凭借此可以快速判断哪些不是广度优先遍历。
#知识点 适用于压缩存储稀疏矩阵的存储结构：三元组表、十字链表
